승격 트리거 - Markdown 노트 (Promote 패턴)
--------------------------------------------------------------------------------

Canvas/PDF 승격은 해당 섹션 참조

본문 편집
  ☐ Preview 탭에서 첫 글자 입력 → Permanent로 승격
  ☐ 승격 후 탭 제목이 일반체(non-italic)로 변경 확인

인라인 제목 편집 (300ms debounce)
  ☐ Preview 탭에서 인라인 제목 수정 → 300ms 후 Permanent로 승격
  ☐ 빠른 연속 수정 시 300ms 후 1회만 승격 확인

파일 이름 변경
  ☐ Preview 탭 파일의 이름을 탐색기에서 변경 → Permanent로 승격

탭 헤더 더블클릭
  ☐ Preview 탭 헤더 더블클릭 → Permanent로 승격


전역 규칙
--------------------------------------------------------------------------------

패널당 Preview 1개
  ☐ Permanent 탭에서 파일 A 싱글클릭 → Preview 생성
  ☐ 계속해서 파일 B 싱글클릭 → 기존 Preview 재사용 (파일 A → B로 교체)
  ☐ Preview + Empty 상태에서 새 파일 싱글클릭 → 빈 탭보다 Preview 우선 재사용

중복 열기 방지
  ☐ 같은 파일 재클릭 → 포커스 이동 (새 탭 생성 안 됨)
  ☐ 그래프뷰 재실행 → 포커스 이동 (새 탭 생성 안 됨)
  ☐ 다른 패널에서 같은 파일 열기 → 정상 동작 (패널별 독립)

Modifier Key 동작
  ☐ Ctrl/Cmd + 파일 탐색기 클릭 → 새 Permanent 탭 생성
  ☐ Ctrl/Cmd + 북마크 클릭 → 새 Permanent 탭 생성
  ☐ Ctrl/Cmd + 검색 결과 클릭 → 새 Permanent 탭 생성

포커스 관리
  ☐ 새 탭 생성 시 해당 탭으로 포커스 이동
  ☐ 탭 닫기 시 탐색기 선택 상태 정리 (is-active, has-focus 제거)

알려진 버그 목록
--------------------------------------------------------------------------------

1. 더블클릭 → 탭닫기 → 싱글클릭 문제 (미해결)
--------------------------------------------------------------------------------

현상:
  1. 파일 탐색기에서 파일을 더블클릭으로 열기
  2. 탐색기에 is-active(회색 블록) + has-focus(진회색 테두리) 표시됨
  3. 아무 동작 없이 바로 탭을 닫기
  4. 탐색기에 is-active와 has-focus가 잔류함
  5. 이 상태에서 해당 파일을 싱글/더블클릭해도 아무 반응 없음 (이미 활성으로 인식)

임시 해결:
  - 다른 파일을 한번 클릭해서 활성 해제하면 다시 정상 동작

재현 절차:
  1. 파일 탐색기에서 "test.md" 더블클릭
  2. "test.md"가 Permanent 탭으로 열림
  3. 탐색기에서 "test.md"에 is-active + has-focus 표시 확인
  4. 즉시 탭 닫기 (×) 클릭
  5. 탐색기에서 "test.md"에 여전히 is-active + has-focus 표시됨
  6. "test.md" 다시 클릭 → 반응 없음

예상 원인:
  - WorkspaceLeaf.detach 패치에서 clearAllSidebarSelections() 호출하고 있으나 완전히 해결되지 않음
  - 더블클릭 시 설정된 상태가 탭 닫기 시 올바르게 정리되지 않음
  - Obsidian 내부 상태와 DOM 상태 불일치

관련 코드:
  - main.ts:948-967 (WorkspaceLeaf.detach 패치)
  - main.ts:1006-1049 (clearAllSidebarSelections)
  - main.ts:1051-1054 (clearExplorerSelection)

해결 방안 검토 필요:
  ☐ 더블클릭 시점의 상태 추적 및 분석
  ☐ detach 시점에 추가 정리 로직 필요 여부 확인
  ☐ Obsidian 이벤트 순서 재검토


2. Daily Notes 판별 패턴 개선 필요 (부분 해결)

현상:
  - Daily Notes 생성 시 파일이름변경 모달 표시 (버그 #2)
